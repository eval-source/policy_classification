# Data sources & licenses

This repo publishes its datasets under `data/<domain>/` (synthetic splits) and `data/<domain>/real/`
(plus `data/real/` for IT). Two kinds of data:

- **Synthetic** (`data/<domain>/{train,val,test}.jsonl`): generated by our own seeded generators
  (`data/<domain>/generate.py`). Ours to share. The **IT** synthetic set deliberately contains
  **fake, non-functional secret-format strings** (AWS-key/GitHub-token/JWT shapes with random
  values) so the classifier learns to detect credential *patterns* ‚Äî they are not real secrets.
- **Real** (`data/<domain>/real/`, `data/real/`): pulled from **open Hugging Face datasets** via the
  committed fetchers (`data/<domain>/fetch_real.py`). We redistribute small derived subsets here for
  convenience and reproducibility; **each remains under its upstream license** ‚Äî verify before reuse,
  especially for commercial use. Credits below.

> **Not published:** `results/label_audit.jsonl` (it captured a real disclosed password during the IT
> label audit), the Tinker API key (`.env`), per-example prediction dumps, and the co-evolution
> anchor/pool artifacts. All are reproducible from the committed code.

## Real-data sources by domain

**IT** (`data/real/`, via `data/it/fetch_real.py`)
- `ai4privacy/pii-masking-300k` ‚Äî PII handling (the dataset's PII is **synthetic/generated**, not real individuals)
- `LLM-PBE/enron-email` ‚Äî corporate email (the **Enron corpus is public-domain**, FERC release)
- `Tobi-Bueck/customer-support-tickets` ‚Äî support chatter / incident tickets
- `iamtarun/python_code_instructions_18k_alpaca`, `Nan-Do/code-search-net-python`, `google/code_x_glue_ct_code_to_text` ‚Äî public code (negatives; secret-regex overrides)
- `galcan/terraform_sec` ‚Äî infrastructure-as-code
- `davidquicast/my-information-security-policy-distiset` ‚Äî infosec policy docs
- `AlicanKiraz0/All-CVE-Records-Training-Dataset` ‚Äî CVE advisories (dropped from the benchmark; kept for revisit)

**Legal** (`data/legal/real/`, via `data/legal/fetch_real.py`)
- `coastalcph/lex_glue` ‚Äî `ledgar` (contract provisions, positives) and `unfair_tos` (ToS sentences, hard negatives)
- `dvgodoy/CUAD_v1_Contract_Understanding_clause_classification` ‚Äî CUAD contract clauses (CC-BY 4.0; cross-source positives)
- `nguha/legalbench` (`privacy_policy_qa`) ‚Äî privacy-policy prose (hard negatives)
- `jonathanli/legal-advice-reddit` ‚Äî r/legaladvice commentary (hard negatives)
- `FiscalNote/billsum` ‚Äî US legislation (rubric-call negatives)
- `fancyzhx/ag_news` ‚Äî general news (easy negatives)

**Marketing** (`data/marketing/real/`, via `data/marketing/fetch_real.py`)
- `ckandemir/amazon-products`, `bprateek/amazon_product_description` ‚Äî product marketing copy (positives)
- `mteb/amazon_polarity`, `cornell-movie-review-data/rotten_tomatoes` ‚Äî customer/movie reviews (opinion negatives)
- `sentence-transformers/wikipedia-en-sentences` ‚Äî encyclopedic prose (negatives)
- `mteb/tweet_sentiment_extraction` ‚Äî tweets (casual negatives)
- `fancyzhx/ag_news` ‚Äî general news (easy negatives)

## Reproduce
```bash
python data/<domain>/generate.py --variant v1 --seed 0 --out data/<domain>   # synthetic
python data/<domain>/fetch_real.py --limit 500 --seed 0                       # real (re-pulls from HF)
```
